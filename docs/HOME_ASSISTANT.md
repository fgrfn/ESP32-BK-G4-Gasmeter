# Home Assistant

## MQTT discovery

The firmware creates one Home Assistant device per ESP32. The default MQTT base topic includes the device ID:

```text
gasmeter/esp32_gas_aabbcc/state
gasmeter/esp32_gas_aabbcc/diagnostics
gasmeter/esp32_gas_aabbcc/availability
```

Entities include total and period volume/energy, flow, Wi-Fi and M-Bus diagnostics, time synchronization, continuous-flow warning and optional configuration controls.

Enable `MQTT commands` only when the broker and Home Assistant installation are trusted. It exposes poll/restart buttons and number entities for measurement settings.

## Gas costs

Cost calculations are intentionally not part of the firmware. Prices, taxes, base fees and contract periods are better maintained centrally in Home Assistant.

Example template using the daily energy entity and a helper containing the work price in EUR/kWh:

```yaml
template:
  - sensor:
      - name: "Gaskosten heute"
        unique_id: gas_cost_today
        unit_of_measurement: "€"
        device_class: monetary
        state: >
          {{
            (
              states('sensor.esp32_gas_aabbcc_gas_energy_today') | float(0)
              * states('input_number.gas_price_eur_per_kwh') | float(0)
            ) | round(2)
          }}
```

A monthly base fee can be added separately or apportioned per day. Replace the entity IDs with the IDs generated by your Home Assistant instance.

## Continuous-flow warning

The warning is a plausibility indicator only. It becomes active when calculated flow remains above the configured threshold for the configured duration. Meter resolution and polling intervals affect detection speed.

Do not use this entity as a replacement for a certified gas detector, emergency shutoff or professional leak inspection.

## Removing stale entities

Firmware 3.1 publishes empty retained discovery payloads for the removed cost entities and for command entities when MQTT commands are disabled. Home Assistant should remove them automatically after the next MQTT connection. Manually delete an entity only when an old retained message from a different base topic remains on the broker.
